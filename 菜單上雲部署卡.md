## 步驟一：跑 SQL（一次性，2 分鐘）

1. 開 [Supabase Dashboard](https://supabase.com/dashboard) → 專案 `laolu-order` → 左邊 **SQL Editor**
2. 開專案資料夾裡的 `setup-menu.sql`，**整份複製貼上 → Run**
3. 看到 `Success. No rows returned` 就完成
4. 驗一下：左邊 **Table Editor** → 應該多了 `menu_items` 表，裡面 **76 個品項**（重跑也不會灌成兩份，放心）

## 步驟二：UAT 驗收清單（照勾）

準備：電腦開看單台（登入老闆帳號）＋手機開點餐頁（用 4G，別連家裡 WiFi 才是真實環境）。

- [ ] **看單台**工具列多了「🍢 菜單」按鈕 → 點開看到全部品項分類陳列
- [ ] 隨便挑一項（例如「蘿蔔」）按成 **❌ 已售完**
- [ ] **手機點餐頁**：10 秒內（或切出去再切回來）「蘿蔔」變灰＋標「今日售完」＋「＋」按不了
- [ ] **時間差攻防**：手機先把某品項加進購物車 → 電腦才按它售完 → 手機直接送出訂單 → 應該被擋：「⚠️ ○○ 剛剛售完了」，訂單**沒有**送進看單台
- [ ] 任選組合區：售完的品項也變灰不能選（已選的可以按「－」退掉）
- [ ] 看單台按回 **✅ 供應中** → 手機 10 秒內恢復可點
- [ ] **黑箱驗收（RLS 門禁）**：開一個**無痕視窗**（＝沒登入的路人身分），隨便開一頁按 F12 進 Console，貼這段假裝駭客改菜單：
