## 步驟一：跑 SQL（2 分鐘）

1. 開 `setup-order-edit.sql`，把 **三處**【WEBHOOK_SECRET】換成真通行碼
   （在你的 `laolu-line-keys.txt`；跟 M1 那次同一串，注意不能帶換行）
2. Supabase → SQL Editor → 新分頁貼上 → **Run** → `Success. No rows returned`
3. **跑完把檔案裡的真值改回【WEBHOOK_SECRET】再存檔**（公開 repo，真值不進 GitHub）
4. 這份 SQL 蓋了：門鈴加「訂單內容更新」鈴聲＋ `orders` 表開 Realtime 廣播

## 步驟二：重部署推播員（3 分鐘）

1. Supabase → **Edge Functions** → `line-push` → 開編輯器
2. 把專案裡 `supabase/functions/line-push/index.ts` 的**新版全文**貼上覆蓋 → **Deploy**
3. 新版多了什麼：`kind = 'edited'` → 發「✏️ 訂單內容更新」給綁定 LINE 的客人
   （Secrets 不用動，沿用原本的 LINE_CHANNEL_ACCESS_TOKEN／WEBHOOK_SECRET）

## 步驟三：UAT 驗收清單

- [ ] 看單台「🆕 新單」「🔥 製作中」的訂單卡多了「**✏️ 編輯**」按鈕（已完成/取消的沒有）
- [ ] 編輯彈窗：**−／＋改量**、**✕ 刪品項**、**下拉＋加入**（售完的品項不在下拉裡）、合計即時重算＋顯示（原 $XX）
- [ ] 按「預覽變更」→ 看到客人訊息：新內容＋合計對照＋（製作中的單）取餐時間不變
- [ ] 預覽按「**↩ 返回**」→ 回編輯彈窗，剛改的都還在；再從編輯按「↩ 返回」→ 原單完全沒動
- [ ] 按「**知道了**」→ 卡片內容立刻更新成新品項新金額
- [ ] **LINE 實測**：用你綁定過的測試單編輯一次 → 你的 LINE 收到「✏️ 訂單內容更新」
